## 产品概述

### 产品名称
**公众号发布助手** (WeChat Publisher)

### 产品定位
一款飞书多维表格边栏插件，支持将多维表格中存储的 Markdown 文章一键排版，既可复制粘贴，也可通过 API 发布到公众号（默认草稿箱，可选直接发布）。

### 核心价值
- **一站式管理**：在多维表格中统一管理文章内容、发布状态、发布时间
- **精美排版**：内置多款主题，自动将 Markdown 转换为公众号适配的富文本格式
- **高效发布**：既可复制粘贴，也可一键发布（草稿箱或直接发布）
- **批量操作**：支持批量选择文章进行发布

---

## 背景与目标

### 业务背景
1. 内容创作者使用 Markdown 编写文章，但公众号不支持直接粘贴 Markdown
2. 排版工具分散，需要在多个平台间切换
3. 文章管理混乱，难以追踪发布状态

### 产品目标

| 目标         | 衡量指标         | 目标值  |
| ------------ | ---------------- | ------- |
| 提升发布效率 | 单篇文章发布耗时 | < 30 秒 |
| 简化操作流程 | 发布所需步骤数   | ≤ 3 步  |
| 统一内容管理 | 用户满意度       | > 4.5/5 |

## 用户画像

### 目标用户
1. **自媒体运营者**
   - 日常使用多维表格管理内容选题和排期
   - 需要高频发布公众号文章
   - 追求高效的工作流

2. **企业内容团队**
   - 多人协作管理公众号内容
   - 需要统一的排版规范
   - 需要内容审批和状态追踪

3. **技术博主/知识分享者**
   - 习惯使用 Markdown 写作
   - 追求精美的代码高亮和排版
   - 可能同时发布多个平台

### 使用场景
1. 在多维表格中完成文章撰写后，直接发布到公众号
2. 从多维表格批量选择已审核的文章进行发布
3. 使用统一的主题风格保持品牌一致性

### 功能详细说明

#### 1. 公众号配置

**功能描述**：用户首次使用时需要配置微信公众号的 API 凭证

**配置项**：
| 配置项     | 必填 | 说明                         |
| ---------- | ---- | ---------------------------- |
| AppID      | ✅    | 公众号的 App ID              |
| AppSecret  | ✅    | 公众号的 App Secret          |
| 公众号名称 | ❌    | 用于展示，方便管理多个公众号 |

**验证流程**：
1. 用户输入 AppID 和 AppSecret
2. 系统调用微信 API 获取 access_token
3. 验证成功：保存配置，显示公众号信息
4. 验证失败：提示错误原因（IP白名单、凭证错误等）

**注意事项**：
- AppSecret 本地保存（用户设备），发布时每次随请求发送，服务端不落库
- 提供"一键清除/重新填写"入口
- 需要提醒用户配置 IP 白名单

**AppSecret 存储安全设计**：

| 存储位置     | 方式                    | 安全性 | 说明                           |
| ------------ | ----------------------- | ------ | ------------------------------ |
| 浏览器端     | localStorage            | 中     | 仅当前设备可访问               |
| 传输过程     | HTTPS 加密              | 高     | 全程 TLS 加密传输              |
| 服务端       | **不存储**              | -      | 用完即弃，不落库不缓存         |

**安全措施**：
1. **本地存储隔离**：使用飞书插件的存储 API（如有）或 localStorage，数据仅存在用户设备
2. **传输加密**：所有 API 请求强制 HTTPS
3. **服务端无状态**：AppSecret 仅用于获取 access_token，获取后立即丢弃
4. **一键清除**：提供明显的「清除配置」入口，用户可随时删除本地凭证
5. **安全提示**：首次配置时展示安全说明，提醒用户保护好 AppSecret

**安全提示文案**：
```
⚠️ 安全提示

• AppSecret 是公众号的核心凭证，请勿泄露
• 您的 AppSecret 仅保存在本设备，不会上传到服务器
• 发布时会通过加密通道传输，用完即弃
• 如需更换设备，请在此设备清除配置后重新输入

[我已了解]
```

#### 4. 发布流程

##### 📌 两种发布方案对比

经过调研，我们提供 **两种发布方式**，用户可根据自身情况选择：

| 维度           | 🅰️ 复制粘贴方式                       | 🅱️ API 自动发布                      |
| -------------- | ------------------------------------ | ----------------------------------- |
| **操作方式**   | 复制排版后的内容，粘贴到公众号编辑器 | 一键发布到草稿箱/直接发布           |
| **配置难度**   | ⭐ 零配置                             | ⭐⭐⭐ 需配置 AppID/AppSecret/IP白名单 |
| **图片处理**   | ✅ 图片可随内容一起粘贴               | ✅ 自动上传到微信素材库              |
| **公众号要求** | ❌ 无需认证                           | ⚠️ 需要已认证的公众号                |
| **发布结果**   | 手动发布（在公众号后台）             | 草稿箱（默认）/直接发布（可选）     |
| **适用场景**   | 个人用户、未认证公众号               | 企业用户、高频发布、自动化需求      |
| **自动化程度** | 手动粘贴                             | 全自动                              |
| **推荐优先级** | ⭐⭐⭐ MVP 必做                         | ⭐⭐⭐ MVP 必做                        |

##### 🅰️ 方案一：复制粘贴方式（MVP 必做）

**优势**：
- ✅ **零配置**：无需 AppID、AppSecret、IP 白名单
- ✅ **无需认证**：个人订阅号也可使用
- ✅ **图片无忧**：富文本复制时图片会一起携带
- ✅ **开发简单**：纯前端实现，无需后端服务

**发布流程**：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  选择文章     │ ──▶ │  预览排版    │ ──▶ │  复制内容    │ ──▶ │  粘贴发布    │
│  (从表格)    │     │  (选择主题)  │     │  (一键复制)  │     │  (公众号后台) │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
```

**技术实现**：
1. 使用 `huasheng_editor` 的 Markdown-it + 主题样式渲染 Markdown
2. 生成带内联样式的 HTML（公众号兼容格式）
3. 复制前将图片转为 Base64，减少公众号粘贴丢图问题
4. 使用 Clipboard API 复制富文本内容到剪贴板
5. 用户粘贴到公众号编辑器

**关键代码思路**：
```javascript
// 复制富文本到剪贴板
async function copyRichTextToClipboard(html) {
  const blob = new Blob([html], { type: 'text/html' });
  const clipboardItem = new ClipboardItem({ 'text/html': blob });
  await navigator.clipboard.write([clipboardItem]);
}
```

**优势**：
- ✅ **全自动**：一键发布，无需手动操作
- ✅ **可回写**：自动更新发布状态和草稿/发布 ID
- ✅ **批量支持**：支持批量发布多篇文章
- ✅ **适合企业**：可集成到自动化工作流
- ✅ **发布灵活**：可选草稿箱或直接发布

**限制条件**：
- ⚠️ 需要已认证的公众号
- ⚠️ 需要配置 IP 白名单
- ⚠️ 需要后端服务支持
- ⚠️ 直接发布需开通发布接口权限，默认发布到草稿箱

**发布流程**：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  选择文章     │ ──▶ │  预览排版    │ ──▶ │  一键发布    │ ──▶ │  自动回写    │
│  (从表格)    │     │  (选择主题)  │     │  (API调用)   │     │  (状态更新)  │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
```

**数据流程**：

```
1. 读取记录数据
       │
       ▼
2. 提取正文图片 ──▶ 上传到微信素材库 ──▶ 获取 media_id
       │                                         │
       ▼                                         │
3. Markdown 解析 & 排版                           │
       │                                         │
       ▼                                         │
4. 生成 HTML 内容 ◀─────── 替换图片 URL ◀────────┘
       │
       ▼

```

#### 5. 批量发布

**功能描述**：支持选择多条记录批量发布到公众号

**错误处理策略**：
- **继续执行**：单篇文章发布失败时，不中断批量任务，继续处理后续文章
- **逐条回写**：每篇文章发布完成后立即回写状态，不等待批量任务结束
- **结果汇总**：批量任务完成后，展示统计结果

**批量发布流程**：
```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
```

### 图片处理策略

#### 微信素材库限制

| 限制项       | 限制值                        | 说明                         |
| ------------ | ----------------------------- | ---------------------------- |
| 图片大小     | ≤ 10 MB                       | 单张图片最大 10MB            |
| 图片格式     | JPG / PNG / GIF               | 不支持 WebP、SVG 等格式      |
| 图片尺寸     | 无硬性限制                    | 建议宽度 ≤ 1920px            |
| 单次上传数量 | 1 张                          | 需逐张上传                   |

#### 自动压缩策略

为确保图片能成功上传，系统自动执行以下处理：

```
原始图片
    │
    ▼
1. 格式检测
    ├── WebP/其他 ──▶ 转换为 JPG
    └── JPG/PNG/GIF ──▶ 保持原格式
    │
    ▼
2. 尺寸检测
    ├── 宽度 > 1920px ──▶ 等比缩放至 1920px
    └── 宽度 ≤ 1920px ──▶ 保持原尺寸
    │
    ▼
3. 大小检测
    ├── > 10MB ──▶ 质量压缩（0.9 → 0.8 → 0.7 递减直到 < 10MB）
    └── ≤ 10MB ──▶ 保持原质量
    │
    ▼
上传到微信素材库
```

**压缩参数**：
| 参数        | 默认值  | 说明                           |
| ----------- | ------- | ------------------------------ |
| maxWidth    | 1920    | 最大宽度（px）                 |
| maxHeight   | 1920    | 最大高度（px）                 |
| quality     | 0.85    | 初始压缩质量                   |
| minQuality  | 0.6     | 最低压缩质量                   |
| maxFileSize | 10MB    | 目标文件大小上限               |

**压缩失败处理**：
- 如压缩后仍超过 10MB，提示用户手动处理
- 如格式转换失败，提示用户更换图片格式

### 安全设计

#### 1. API 安全

| 安全措施         | 说明                           |
| ---------------- | ------------------------------ |
| **HTTPS**        | 使用 Let's Encrypt 免费证书    |
| **CORS**         | 限制只允许多维表格插件域名访问 |
| **请求频率限制** | 防止滥用，限制每分钟请求数     |
| **参数校验**     | 严格校验输入参数               |

#### 2. 敏感信息处理

| 数据         | 处理方式                                       |
| ------------ | ---------------------------------------------- |
| AppSecret    | 本地保存（用户设备），发布时传输，服务端不存储 |
| access_token | 内存缓存，按公众号隔离，定期刷新               |
| 请求日志     | 脱敏处理敏感字段                               |

#### 3. access_token 管理

**微信 access_token 特性**：
- 有效期：7200 秒（2 小时）
- 每日获取上限：2000 次
- 同一公众号同一时间只有一个有效 token

**服务端管理策略**：

```
请求发布 API
    │
    ▼
检查内存缓存是否有该 AppID 的 token
    │
    ├── 有且未过期 ──▶ 直接使用
    │
    └── 无或已过期 ──▶ 使用 AppID + AppSecret 获取新 token
                            │
                            ▼
                      缓存到内存（按 AppID 隔离）
                            │
                            ▼
                      设置过期时间（提前 5 分钟刷新）
```

**缓存结构**：
```typescript
// 内存缓存（Map 结构）
interface TokenCache {
  [appId: string]: {
    accessToken: string;
    expiresAt: number;  // 过期时间戳
  }
}
```

**注意事项**：
- 服务端不持久化 token，重启后重新获取
- 多用户共用同一公众号时，共享同一个 token
- 提前 5 分钟刷新，避免请求时刚好过期
- 如获取 token 失败，返回明确错误信息（如 IP 白名单未配置）

文章内容区域支持 **编辑模式** 和 **预览模式** 两种视图，在同一区域内切换：

**切换方式**：

| 交互方式     | 操作                                                 | 说明                      |

**交互细节**：

- **编辑同步**：编辑模式下修改内容后，切换到预览模式自动渲染最新效果

- **内容回写**：编辑后的内容可选择回写到多维表格（P1 功能）
- **滚动位置**：切换模式时尽量保持滚动位置一致
- **发布方式**：草稿箱/直接发布开关，默认草稿箱
- **编辑器特性**：
  - 语法高亮显示 Markdown 格式

  - 显示行号（可选）

### 发布权限说明

**草稿箱 vs 直接发布** 权限对比：

| 功能       | API 接口             | 权限要求                     | 说明                       |
| ---------- | -------------------- | ---------------------------- | -------------------------- |
| 新增草稿   | `POST /cgi-bin/draft/add` | 已认证公众号                 | 所有认证号均可使用         |
| 直接发布   | `POST /cgi-bin/freepublish/submit` | 已认证 + 发布权限            | 需单独申请发布接口权限     |

**直接发布权限申请**：
1. 登录微信公众平台
2. 进入：设置与开发 → 接口权限
3. 找到「发布能力」，点击申请
4. 审核通过后方可使用直接发布功能

**用户引导策略**：
- 默认使用「草稿箱」模式，降低使用门槛
- 选择「直接发布」时，检测权限是否开通
- 如权限未开通，提示用户：
  ```
  ⚠️ 您的公众号尚未开通发布接口权限

  直接发布需要在微信公众平台申请「发布能力」。

  您可以：
  • 前往公众平台申请权限
  • 或先发布到草稿箱，再到公众号后台手动发布

  [申请权限] [使用草稿箱]
  ```

**2. 内联样式应用**（参考 `app.js` 第 1014-1088 行）：
```javascript
applyInlineStyles(html) {
  const style = STYLES[this.currentStyle].styles;
  const parser = new DOMParser();
  const doc = parser.parseFromString(html, 'text/html');

  // 遍历每种元素类型，应用对应样式
  Object.keys(style).forEach(selector => {
    const elements = doc.querySelectorAll(selector);
    elements.forEach(el => {
      const currentStyle = el.getAttribute('style') || '';
      el.setAttribute('style', currentStyle + '; ' + style[selector]);
    });
  });

  // 包装容器
  const container = doc.createElement('div');
  container.setAttribute('style', style.container);
  container.innerHTML = doc.body.innerHTML;

  return container.outerHTML;
}
```

**3. 图片存储**（参考 `app.js` 第 9-200 行）：
- 使用 `IndexedDB` 存储压缩后的图片
- 复制时从 IndexedDB 读取 Blob 转 Base64
- 支持图片压缩配置（maxWidth、maxHeight、quality）

1. **直接复用 `styles.js`**：可以直接复制 `styles.js` 到项目中使用
2. **自实现 `applyInlineStyles`**：参考上述代码逻辑，在 React 项目中实现
3. **图片处理参考**：IndexedDB + Canvas 压缩方案可直接参考
